# 스마트공장 웨어러블 기반 무작업(Hands-free) 조치보고
## 웹 대시보드 인터랙티브 시연 시나리오 5종 (회의 공유용)

> 목적: 현장(웨어러블/모바일)에서는 **타이핑 없이** 조치 이벤트·증거를 남기고,  
> 웹에서는 **상태 전환/원격지원/보고서 큐레이션/리플레이/격리**를 “조작”하며 의사결정하는 흐름을 시연한다.

---

## 0) 공통 전제(시연 기준)

- **현장 디바이스**: 웨어러블이 없으면 스마트폰으로 대체 가능(카메라/마이크/버튼 이벤트).
- **웹 대시보드**: 실시간 반영(예: WebSocket/SSE 느낌)이 핵심. “바뀌는 화면”이 데모의 가치.
- **데이터**: MES/센서/LOT는 더미로도 충분. 중요한 건 **연결된 UX**.
- **핵심 메시지**: 보고서 ‘작성’이 아니라 **웹이 사건을 ‘진행 화면 + 보고서’로 자동 구성**한다.

---

## 1) 역할(권한) 정의

- **작업자(현장)**: 조치 시작/종료, 체크리스트 응답, 사진/영상 업로드, 음성 메모
- **관제(모니터링)**: 알람 확인, 인시던트 상태 전환, 담당자 할당/재할당, 원격지원 호출
- **전문가(기술지원)**: 라이브 영상 주석, 작업 스텝 가이드 작성, 검증 코멘트
- **품질(QA)**: LOT 영향 범위 선택, 격리/재검사 태스크 배포, 해제 승인
- **승인자(관리자)**: 보고서 승인/반려, CAPA 확정

---

## 2) 공통 데이터/이벤트(웹 인터랙션을 살리는 최소 단위)

### 2-1. 공통 오브젝트
- **Alarm**: 발생시각, 심각도, 라인, 설비ID, 증상코드, 센서 스냅샷(더미 가능)
- **Incident**: 상태, 담당자, SLA 타임라인, 관련 알람/설비/LOT
- **Action**: 체크리스트 항목, 수행시각, 수행자, 결과(성공/보류/실패)
- **Evidence**: 사진/영상/음성전사/센서스냅샷/주석스냅샷, 태그, 시간코드
- **Task**: 재검사/부품교체/현장확인 지시, 기한, 담당자

### 2-2. 공통 이벤트(데모에서 화면이 “움직이게” 만드는 트리거)
- `ALARM_RAISED` (알람 발생)
- `INCIDENT_CREATED` (인시던트 생성)
- `ASSIGNED / REASSIGNED` (담당자 할당/변경)
- `DISPATCHED` (출동 시작)
- `ARRIVED_ON_SITE` (현장 도착)
- `ACTION_STARTED / ACTION_COMPLETED` (조치 시작/완료)
- `EVIDENCE_UPLOADED` (증거 업로드)
- `STEP_CONFIRMED` (원격지원 스텝 확인)
- `VERIFICATION_PASSED / FAILED` (검증 결과)
- `INCIDENT_CLOSED` (종결)
- `LOT_QUARANTINED / RELEASED` (LOT 격리/해제)

> **포인트:** 위 이벤트를 “데모용 버튼”으로 만들어도 충분히 인터랙티브한 시연이 가능.

# 시나리오 1) 실시간 관제보드: “알람 → 출동 → 조치” + SLA 타임라인

## 1-1. 시나리오 요약
알람 발생 → 인시던트 생성 → 웹에서 **카드 이동/담당자 할당** →  
현장 이벤트가 들어올 때마다 **SLA 타임라인 자동 기록** + 상태 변화.

## 1-2. 웹 화면 구성(추천)
- **관제 메인(Operations Board)**
  - 알람 스트림(좌): 최신 알람 카드 + 필터(라인/심각도/설비)
  - 인시던트 칸반(중앙): `신규 → 출동중 → 조치중 → 검증중 → 종료`
  - 미니맵(우상): 라인/설비 도식(가상 위치 표시 OK)
  - SLA 위젯(우하): 평균 대응/지연 인시던트
- **인시던트 상세 패널(드로어)**
  - 상태/담당자 변경, 원격지원 시작
  - SLA 타임라인(마커)
  - 증거 썸네일/코멘트/체크리스트 진행률

## 1-3. 핵심 웹 인터랙션(시연 포인트)
- 칸반 **드래그&드롭**으로 상태 전환
- 담당자 **즉시 재할당**
- 타임라인 마커 클릭 → 해당 시점 증거/전사로 점프
- 증거 업로드 시 카드에 `+1` 뱃지/썸네일 즉시 갱신

## 1-4. 현장 연동(최소)
- 버튼 3개로 충분: `현장도착`, `조치시작`, `조치종료`
- 사진 1장 업로드(또는 더미) → `EVIDENCE_UPLOADED`

## 1-5. 데모 스크립트
1) 웹에서 “알람 발생” → 알람 카드 + 인시던트 생성  
2) 관제자가 카드 “출동중”으로 드래그 + 담당자 지정  
3) (모바일) “현장도착” → SLA 타임라인 마커 생성  
4) (모바일) 사진 업로드 → 카드 썸네일/증거 개수 증가  
5) (모바일) “조치종료” → 상태 “검증중” 이동  
6) 관제 “검증 통과” → 종료 + 보고서 초안 생성 표시

## 1-6. 성공 기준
- [ ] 상태 전환(드래그)이 즉시 반영됨  
- [ ] 현장 이벤트가 타임라인에 즉시 기록됨  
- [ ] 증거 업로드가 카드/상세에 즉시 나타남  
- [ ] 종료 시 “보고서 자동 생성” 느낌이 명확함

# 시나리오 2) 원격지원 룸: 라이브 영상 + 웹 주석(그려서 되돌려주기)

## 2-1. 시나리오 요약
현장 영상 스트리밍 → 웹에서 전문가가 **영상 위 주석(화살표/텍스트)** →  
현장에 오버레이로 전달 → 주석 포함 스냅샷이 **증거로 자동 저장**.

## 2-2. 웹 화면 구성(추천)
- **라이브 뷰어(중앙)**: 지연/프레임/연결상태
- **주석 툴바(상단)**: 펜/원/화살표/텍스트/지우개/되돌리기
- **Freeze(정지) + Zoom/Pan**: 정지 프레임에 주석 고정
- **스텝 가이드(우측)**: Step 1~N 작성/완료 체크
- **증거 타임라인(하단)**: “주석 스냅샷” 썸네일 자동 쌓임

## 2-3. 핵심 웹 인터랙션
- 영상 위 **직접 드로잉** + Freeze로 디테일 표시
- Step 편집 → 현장에는 “다음 스텝”만 크게 노출(Hands-free)

## 2-4. 현장 연동(최소)
- 모바일 카메라 스트림(실제/더미)
- 현장: “확인/다음” 이벤트만 전송

## 2-5. 데모 스크립트
1) 인시던트 → “원격지원 시작”  
2) 영상 Freeze → 화살표 + “재체결” 텍스트  
3) Step 1~2 작성  
4) 현장 “확인” → 웹 Step 완료 반영  
5) 주석 스냅샷이 증거함에 자동 저장됨 확인

## 2-6. 성공 기준
- [ ] 웹 주석이 영상 위에서 자연스럽게 동작  
- [ ] 주석 포함 스냅샷이 자동 저장(증거화)  
- [ ] Step 완료가 양방향 동기화되는 느낌

# 시나리오 3) 보고서 빌더: 증거함 → 섹션 드래그&드롭 + 전사 인용문 생성

## 3-1. 시나리오 요약
웨어러블이 올린 전사/사진/센서 로그가 “증거함”에 모이고,  
웹에서 보고서 섹션으로 **드래그&드롭 큐레이션**하여 보고서를 완성.

## 3-2. 웹 화면 구성(추천)
- **좌측: 보고서 템플릿 섹션**
  - `현상 / 원인추정 / 조치내용 / 검증결과 / 재발방지(CAPA)`
- **우측: 증거함(Inbox)**
  - 음성전사 카드(시간코드)
  - 사진/영상 카드(썸네일)
  - 센서 스냅샷 카드(그래프 미리보기)
- **버전/승인 패널**
  - 변경점 하이라이트(v1↔v2), 승인/반려

## 3-3. 핵심 웹 인터랙션
- 증거카드 → 섹션 드롭 시 자동 레이아웃 배치
- 전사 텍스트 **드래그 하이라이트 → 인용문 블록 생성**
- 센서 스냅샷을 “검증결과”로 드롭하면 **전/후 비교 카드** 자동 생성(추천)

## 3-4. 현장 연동(최소)
- 음성 메모 2개(현상/결과), 사진 1장, 센서 스냅샷(더미) 1개

## 3-5. 데모 스크립트
1) 증거함에 전사 2개 + 사진 1개가 보이게 준비  
2) 전사 문장 드래그 → “현상” 섹션에 인용문 삽입  
3) 사진을 “조치내용”에 드롭 → 썸네일 자동 배치  
4) 센서 스냅샷을 “검증결과”에 드롭 → 전/후 비교 표시  
5) 승인 요청 → 승인자가 승인

## 3-6. 성공 기준
- [ ] “보고서 작성=카드 정리”라는 인식이 생김  
- [ ] 인용문 생성/자동 배치가 직관적  
- [ ] 승인/반려 흐름이 명확

# 시나리오 4) 타임머신 리플레이: 시간 슬라이더로 사건 전후를 재생하며 원인분석

## 4-1. 시나리오 요약
사건 전후 30~60분을 웹에서 **시간 슬라이더로 스크럽**하면  
센서 그래프/이벤트/증거가 동기화되어 원인분석이 쉬워짐.

## 4-2. 웹 화면 구성(추천)
- 상단: 시간 범위(±30/60)
- 중앙: 멀티 트렌드 그래프(진동/온도/전류 등)
- 하단: 이벤트 타임라인 마커(알람, 도착, 커버 오픈 등)
- 우측: 선택 시점 상세(사진/전사/스냅샷)
- Split View: 조치 전 5분 vs 후 5분 비교

## 4-3. 핵심 웹 인터랙션
- 슬라이더 이동 → 그래프 커서/이벤트/증거 패널 동기화
- 마커 클릭 → 해당 시점으로 점프
- 구간 드래그 → 피크/평균/변동 요약 팝업

## 4-4. 현장 연동(최소)
- 타임스탬프 이벤트 2~3개만 있어도 충분(“커버 오픈”, “부품교체 시작” 등)
- 센서 시계열은 더미 가능

## 4-5. 데모 스크립트
1) 인시던트 → “리플레이 보기”  
2) 알람 직전으로 이동 → 상승 구간 보여줌  
3) “현장 도착” 마커 클릭 → 전사/사진 표시  
4) 조치 후 구간으로 이동 → Split View로 안정화 비교  
5) 원인 추정 선택 + CAPA 생성 → 보고서 자동 반영

## 4-6. 성공 기준
- [ ] 슬라이더 조작이 재생/되감기처럼 직관적  
- [ ] 이벤트-증거-그래프 연결이 한 화면에서 이해됨  
- [ ] 분석 결과가 보고서로 자연스럽게 이어짐

# 시나리오 5) 품질 격리(Containment): 공정 맵에서 LOT 영향 선택→격리→태스크 배포

## 5-1. 시나리오 요약
불량 등록 → 공정 맵/WIP에서 영향 LOT을 선택/추천받고  
**격리 보드로 드래그**하여 격리 → 재검사 태스크를 현장에 푸시 → 결과가 실시간 반영.

## 5-2. 웹 화면 구성(추천)
- **공정 맵 + WIP**
  - 공정 박스(전/중/후공정) + 수량
  - 불량 공정 강조
  - 공정 클릭 → LOT 리스트
- **LOT 영향 범위 패널**
  - LOT 선택 → 상/하류 영향 후보 자동 하이라이트
  - 격리 대상 체크
- **Quarantine 보드**
  - `정상 / 보류 / 격리 / 재검사중 / 해제대기`
  - LOT 카드 드래그&드롭
- **태스크 배포**
  - 재검사(샘플 10/전수)/재작업 템플릿, 담당자/기한 지정

## 5-3. 핵심 웹 인터랙션
- 영향 후보 LOT 자동 추천(하이라이트)
- LOT 드래그 → 격리 존 이동 시 상태/색상 즉시 변경
- 현장 완료 이벤트 → LOT 카드에 완료 뱃지 + 해제대기 상태

## 5-4. 현장 연동(최소)
- LOT 스캔/선택 + “재검사 완료” 이벤트
- 검사 사진 1장 업로드(증거 자동 첨부)

## 5-5. 데모 스크립트
1) “불량 등록” → 공정 맵 영향 공정 강조  
2) LOT 선택 → 후보 LOT 자동 하이라이트  
3) 격리 대상 체크 → Quarantine으로 드래그  
4) “재검사 태스크 배포” → 현장에 할당  
5) 현장 완료 이벤트 → 웹 LOT 카드 상태 변경  
6) 품질 승인 “해제 승인” → 정상 복귀 + 보고서 기록

## 5-6. 성공 기준
- [ ] 영향 범위가 공정 맵에서 직관적으로 보임  
- [ ] 격리/해제가 드래그로 ‘조작’되는 느낌  
- [ ] 재검사 태스크/결과가 LOT 카드에 연결됨

## 6) 시연용 더미 데이터 세트(추천)

- 라인: `Press-01 / Assemble-02 / Inspect-01`
- 설비: `Motor-A / Press-3 / Conveyor-2`
- 알람:
  - A1) `Motor-A 진동 High (Sev: High)`
  - A2) `Press-3 온도 Warning (Sev: Mid)`
- 인시던트: `INC-1001 (Motor-A 진동)`
- 증거:
  - 음성전사 2개(현상/조치결과)
  - 사진 1장(교체 전/후 중 택1)
  - 센서 스냅샷 2개(조치 전/후)
